# Changelog - v1.3.0

## [1.3.0] - 2026-04-02

### Added
- **Advanced Resilience with Polly**: Integrated the [Polly](https://github.com/App-vNext/Polly) library to handle transient API failures.
- **Configurable Retry Policies**:
    - **Transient Error Handling**: Automatically retries on network timeouts (`TaskCanceledException`), network errors (`HttpRequestException`), and specific HTTP status codes: `408` (Timeout), `429` (Too Many Requests), `500` (Internal Server Error), `502` (Bad Gateway), `503` (Service Unavailable), and `504` (Gateway Timeout).
    - **Exponential Backoff**: Implemented an exponential backoff strategy (e.g., 2s, 4s, 8s) to avoid overwhelming the server during recovery.
    - **Detailed Retry Logging**: Console output now includes the specific reason for each retry attempt (e.g., specific HTTP error code or exception message).
- **Externalized Retry Configuration**: Added `RetrySettings` to `appsettings.json` to allow users to customize `MaxRetries` and `BaseDelaySeconds` without recompiling.

### Changed
- Improved memory management in the retry loop by recreating `StringContent` per attempt to avoid `ObjectDisposedException`.

### Fixed
- Prevented script termination on split-second API downtime or transient network glitches.
